QuantumChat / PQC Benchmark

Security evaluation of Hybrid ML-KEM-768 + ECDH-P256 + AES-256-GCM against classical schemes

● Real WebCrypto ● Real ML-KEM (liboqs) ● Live in browser ⚠ ML-KEM via WASM fallback
What is actually being measured here?

• RSA-2048, ECDH P-256, AES-256-GCM — 100% real operations via the browser's window.crypto.subtle (WebCrypto API). These are native C++ implementations running inside your browser engine. The timings are genuine.

• ML-KEM-768 (Kyber768) — Real ML-KEM-768 algorithm implemented in pure JavaScript, matching the NIST FIPS 203 spec. Key sizes (pk=1184 bytes, ct=1088 bytes) and shared secret correctness are verified. Timing is slightly slower than a native C implementation (like BouncyCastle) due to JS overhead, but the algorithm is real.

• Hybrid handshake — Combines real ECDH key agreement + real ML-KEM encapsulation + real HKDF-SHA256 + real AES-256-GCM. The full pipeline runs end-to-end in your browser.

• Attack demo — The GCM wrong-key rejection is a real crypto.subtle.decrypt call that actually throws a DOMException. Not simulated.
